Get an error when trying to access the vite server through an ngrok tunnel  The error you're seeing is a Mixed Content block. Since you are accessing the Vite frontend via a secure ngrok tunnel (https), 
the browser forbids it from making "insecure" requests to an http backend (your local IP).

1.The Vite Proxy Approach
If you only want to manage one ngrok tunnel, you can tunnel Vite and 
have Vite "proxy" the API requests to FastAPI internally. Since the browser only "talks" to the ngrok URL, it stays in a secure context.


import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'

export default defineConfig({
  plugins: [react()],
  server: {
    allowedHosts: ['all'], // Required to allow ngrok to connect to Vite
    proxy: {
      '/api': {
        target: 'http://127.0.0.1:8001',
        changeOrigin: true,
        // This removes '/api' from the path before sending to FastAPI
        // e.g., /api/login becomes /login
        rewrite: (path) => path.replace(/^\/api/, ''),
      },
    },
  },
})

Instead of calling http://192.168.1.112:8001/login, change your code to call a relative path:
