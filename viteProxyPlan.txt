 Implementation Plan: Enable Vite Proxy for ngrok

 Context

 The user is experiencing "Mixed Content" errors when accessing the Vite frontend via a secure ngrok tunnel (https), because
 the frontend is attempting to make insecure (http) requests to the FastAPI backend. To resolve this, we will implement a
 reverse proxy in Vite. This ensures the browser only communicates with the secure ngrok URL, and Vite handles the
 communication with the backend internally.

 Proposed Changes

 1. Vite Configuration

 Modify frontend/vite.config.ts to add a proxy for API requests.
 - Change: Add server.proxy configuration.
 - Details:
   - Target: http://127.0.0.1:8001
   - Path: /api
   - changeOrigin: true
   - rewrite: (path) => path.replace(/^\/api/, '')
   - Ensure server.allowedHosts: ['all'] (or true) is set.

 2. Frontend Environment Variables

 Update the API base URL to use the relative proxy path.
 - File: frontend/.env
 - Change: Update VITE_API_BASE_URL=http://192.168.1.112:8001 to VITE_API_BASE_URL=/api.

 3. Frontend Config Fallback

 Update the hardcoded fallback in the configuration file.
 - File: frontend/src/config.ts
 - Change: Update API_BASE_URL fallback from http://192.168.1.112:8001 to /api.

 4. API Client Headers

 Add the required ngrok header to all API requests to bypass the ngrok browser warning.
 - File: frontend/src/lib/api.ts
 - Change: Add headers: { 'ngrok-skip-browser-warning': 'true' } to the axios.create configuration.

 5. Backend CORS

 The current backend configuration in backend/app/main.py uses allow_origins=["*"].
 - Analysis: This is already more permissive than the suggested list in enableViteProxyForNgrok.txt and will allow requests
 from the ngrok tunnel.
 - Recommendation: No immediate change required unless the user wishes to move from a wildcard to a specific allowlist.
